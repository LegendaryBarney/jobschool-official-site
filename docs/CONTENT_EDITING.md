## 1. 準備：第一次登入

### 1.1 登入網址

打開瀏覽器（建議 Chrome 或 Edge），輸入：

```
https://jobsedu.com.tw/admin
```

![登入畫面](./images/decap-login.png)

> （工程師會在交付時補上實際截圖）

### 1.2 兩種登入方式（您只會看到其中一種，依工程師最終設定為準）

#### 方案 A：Netlify Identity 登入（建議給業主用，較簡單）

1. 第一次使用，會收到工程師寄給您的「邀請信」（寄件人為 Netlify）
2. 點開信中按鈕「Accept Invite」，會跳到一個設定密碼頁
3. 設定密碼（**請用 1Password / Google 密碼管理器存下來，不要寫便利貼**）
4. 之後每次登入：直接點 `/admin` 頁面的「Login」按鈕，輸入 email + 密碼
5. 忘記密碼：點「Forgot password」，照信操作即可

#### 方案 B：GitHub OAuth 登入

1. 您必須已有 GitHub 帳號（工程師會先把您加為 repo 的 collaborator）
2. 點 `/admin` 頁面的「Login with GitHub」
3. 第一次會跳到 GitHub 授權頁，點「Authorize」
4. 之後就會自動登入，不必再輸入密碼

### 1.3 登入後看到什麼

成功登入後，畫面分為三區：

- **左側**：集合清單（師資、課程、部落格文章…）
- **中間**：選定集合的內容列表
- **右側**：編輯時顯示欄位 + 即時預覽

## 2. 介面導覽

### 2.1 左側集合清單對照

| 後臺集合名稱 | 對應網站位置 | 您會在這裡做什麼 |
|---|---|---|
| **師資** | `/teachers` 與 `/teachers/[名字]` | 新增老師、改學歷年資、換照片 |
| **課程** | `/courses` 與 `/courses/[名稱]` | 新增課程、調價、改上課時段 |
| **部落格文章** | `/posts` 與 `/posts/[文章]` | 寫教學文、考前提醒、班務公告 |
| **學生見證** | `/testimonials` | 新增學生心得、家長回饋 |
| **常見問題** | `/faq` | 新增 FAQ 條目（會自動套 FAQ Schema 給 Google） |
| **活動 Landing Page** | `/lp/[活動代號]` | 寒暑假班、學測衝刺等檔期 LP |

### 2.2 上方工具列

- **Quick Add**：右上角的「+」按鈕，可快速新增任一集合的項目
- **個人選單**：右上角頭像，可登出、切換語言
- **Workflow**：若啟用了 editorial workflow，這裡顯示「草稿 / 待審 / 已發佈」三欄看板

### 2.3 編輯介面

- 左半邊：欄位輸入區（標題、摘要、內文、日期…）
- 右半邊：即時預覽
- 底部：「儲存」「儲存為草稿」「發佈」按鈕

> **重要**：每次改完一定要按「儲存」或「發佈」。瀏覽器關掉前沒按，改的東西會不見。

## 3. 編輯各類內容

以下每種內容都有「**新增**」與「**修改既有**」兩種操作。先示範新增（流程較完整），修改流程在每節最後補一段。

---

### 3.1 新增一篇部落格文章

**情境舉例**：要寫一篇「113 年學測數學考前 5 題型總整理」。

#### 步驟

1. 左側點 **部落格文章**
2. 點右上角 **「新增 部落格文章」** 按鈕
3. 依序填以下欄位：

   | 欄位 | 填什麼 | 範例 |
   |---|---|---|
   | **標題** | 文章標題（會出現在頁面 H1 與瀏覽器分頁） | `113 學測數學考前 5 題型總整理` |
   | **TL;DR / 摘要** | 一段話摘要，2-3 句，會顯示在文章開頭與 Google 搜尋結果 | `學測倒數 30 天，整理 5 個高頻題型...` |
   | **發佈日** | 文章發佈日期（影響排序） | 點日期選擇器選今天 |
   | **最後更新** | （選填）內容大改時更新此欄，網站會顯示「最後更新於 X 月 X 日」 | 留空或選日期 |
   | **作者** | 預設「賈伯斯數理教室」，老師個人發文可填名字 | 留預設 |
   | **分類** | 主分類（如「考試準備」「教學心得」「班務公告」） | `考試準備` |
   | **標籤** | 多個關鍵字，按 Enter 加新標籤 | `學測`、`數學`、`高三` |
   | **封面圖** | 點 Choose an image → 上傳或選 media library | 1200x630px JPG（見 §5） |
   | **草稿** | 開啟 = 不公開；關閉 = 對外可見 | 寫到一半就先開啟，寫完關閉 |
   | **內文** | 文章主體，富文字編輯器 | 見下方 |

4. 在「**內文**」區塊撰寫文章。可用上方工具列加標題、粗體、清單、連結、圖片、表格。Markdown 語法見 [§4](#4-markdown-語法快速指引)。

5. 寫到任一階段，按底部 **「儲存」** 即會存為草稿（不對外公開）。

6. 寫完想公開：
   - 若有 editorial workflow：點 **「Save → 移到 Ready」→「Publish now」**
   - 若無 workflow：直接點 **「Publish」→「Publish Now」**

7. 約 1-2 分鐘後，新文章會自動出現在 `https://jobsedu.com.tw/posts`。

#### 修改既有文章

1. 左側「部落格文章」→ 從列表點開要改的文章
2. 改完按「儲存」或「Publish」（依目前狀態）
3. **建議**：若是大改（如修正錯誤或補充新內容），把「**最後更新**」欄位改成今天

### 3.2 新增師資

**情境舉例**：剛聘了一位新的物理老師「林老師」。

#### 步驟

1. 左側點 **師資** → 右上 **「新增 師資」**
2. 填欄位：

   | 欄位 | 填什麼 | 範例 |
   |---|---|---|
   | **姓名** | 對外稱謂 | `林老師` 或 `林承恩 老師` |
   | **職稱** | 在補習班的角色 | `物理科主任`、`高中物理講師` |
   | **照片** | 半身照，建議 800x800px，正方形 | 上傳照片 |
   | **學歷** | 一行一個，按 Enter 加新項 | `國立台灣大學 物理系`、`國立清華大學 物理研究所碩士` |
   | **教學年資** | 數字，單位「年」 | `8` |
   | **教學科目** | 一行一個 | `高中物理`、`高中化學` |
   | **教學理念** | 2-3 句話的個人簡介 | `物理不是背公式...` |
   | **精選** | 開啟 = 出現在首頁「師資精選」區 | 視情況 |
   | **排序** | 數字越小越前面 | `10` |
   | **SEO 描述** | 給 Google 看的描述（150 字內） | `林老師：台大物理系...` |
   | **內文** | （選填）老師的詳細介紹、得獎紀錄、學生回饋等 | Markdown 撰寫 |

3. 確認後點 **「Publish」**

> **建議**：填「教學理念」時對齊 BRAND tone（溫潤、專業、不浮誇）。可用 §6 速查。

#### 修改既有師資

- 左側「師資」→ 從列表點開要改的老師 → 改完發佈
- 換照片：把舊照片刪掉，再上傳新的

### 3.3 新增課程

**情境舉例**：要新開「高一數學週末班」。

#### 步驟

1. 左側點 **課程** → 右上 **「新增 課程」**
2. 填欄位：

   | 欄位 | 填什麼 | 範例 |
   |---|---|---|
   | **課程名稱** | 完整課程名 | `高一數學 週末加強班` |
   | **年級** | 下拉選單：「國中」或「高中」 | `高中` |
   | **適合年級** | 細分年級，可多選 | `高一` |
   | **科目** | 主科目 | `數學` |
   | **講師（slug）** | 對應的師資英文識別碼（請工程師告知；通常為老師英文名） | `lin-laoshi` |
   | **課程簡介** | 2-3 句簡介 | `針對 108 課綱高一上學期...` |
   | **上課時段** | 一行一個時段 | `每週六 10:00-12:00`、`每週六 14:00-16:00` |
   | **價格區間** | 文字格式 | `每月 NT$ 4,800` 或 `學期 NT$ 18,000` |
   | **封面圖** | 課程主視覺 | 上傳 |
   | **精選** | 開啟 = 首頁顯示 | 視情況 |
   | **排序** | 越小越前 | `20` |
   | **SEO 描述** | 給 Google 看 | 一句話 |
   | **內文** | 完整課程介紹（適合對象、教材樣本、上課流程）| Markdown |

3. 點 **「Publish」**

#### 修改既有課程

- 漲價、改時段：直接改對應欄位 → 發佈
- **建議**：價格 / 時段這類「家長最在意」的資訊改完，當天請自己用手機開一次網站確認顯示正確

### 3.4 新增學生見證

#### 步驟

1. 左側點 **學生見證** → 右上 **「新增 學生見證」**
2. 填欄位：

   | 欄位 | 填什麼 | 範例 |
   |---|---|---|
   | **學生稱謂** | 為保護隱私可只用姓 | `王同學` |
   | **年級** | 該生當時年級 | `高三` |
   | **就讀學校** | 嘉義在地校名（精準對應地方 SEO） | `嘉義高中`、`嘉義女中`、`興華中學`、`協同中學` |
   | **評價（1-5）** | 預設 5 | `5` |
   | **心得引文** | 學生原話，2-4 句 | `從一開始抗拒上補習班...` |
   | **影片網址** | 若有錄影心得，貼 YouTube / Vimeo URL | 留空 |
   | **精選** | 開啟 = 首頁與課程頁推薦 | 視情況 |
   | **排序** | 越小越前 | `0` |
   | **完整內文** | （選填）長版心得 | Markdown |

3. 點 **「Publish」**

> **隱私守則**：上傳前請確認已取得學生與家長的書面同意。建議用「姓 + 同學」即可，避免完整姓名。

### 3.5 新增 FAQ

**情境舉例**：常被家長問「可以單科試聽嗎？」

#### 步驟

1. 左側點 **常見問題** → 右上 **「新增 常見問題」**
2. 填欄位：
   - **問題**：問題本身（會以 H2 顯示） — `可以只試聽單一科目嗎？`
   - **分類**：分類標籤 — `試聽`、`收費`、`課程`
   - **排序**：越小越前 — `10`
   - **答案**：用 markdown 寫，可含清單與連結
3. 點 **「Publish」**

> 此集合會自動產生 FAQ Schema 給 Google，做得好可以在搜尋結果中以「展開式問答」呈現。所以**問題請寫得像家長真的會問的口吻**，不要寫成「關於試聽的說明」這類學究式標題。

### 3.6 新增活動 Landing Page

**情境舉例**：要為「2026 暑期密集班」開一個專屬 LP。

#### 步驟

1. 左側點 **活動 Landing Page** → 右上 **「新增 活動 Landing Page」**
2. 填欄位：

   | 欄位 | 填什麼 | 範例 |
   |---|---|---|
   | **URL slug** | 英數小寫 + 連字符，會出現在網址 | `summer-2026` → URL 為 `/lp/summer-2026` |
   | **頁面標題** | 瀏覽器分頁標題 | `2026 暑期密集班 — 賈伯斯數理教室` |
   | **主標** | LP 第一眼大字 | `40 天，補回一整個學期` |
   | **副標** | 主標下方說明 | `7/15 - 8/25，數理英文小班...` |
   | **主視覺** | hero 圖片（建議 1920x1080） | 上傳 |
   | **CTA 文字** | 按鈕文字 | `立即預約試聽`（預設） |
   | **CTA 連結** | 點按鈕後跳到的頁面 | `/contact` 或 LINE URL |
   | **活動開始日** | 用於倒數 | 選日期 |
   | **活動結束日** | 過期後可選擇下架 | 選日期 |
   | **發佈** | 開啟 = 對外可見 | 視情況 |
   | **內文** | 完整活動說明（課程細節、價格、報名方式）| Markdown |

3. 點 **「Publish」**

> **建議**：活動結束後，把「發佈」關閉即可下架。不必刪除，未來同檔期可複製重用。

## 4. Markdown 語法快速指引

Decap 編輯器多數情況會幫您處理，但若您想直接寫，下表是常用語法。

| 想做什麼 | 語法 | 顯示效果 |
|---|---|---|
| 大標題 H2 | `## 我的標題` | （網頁上的 H2 大標） |
| 中標題 H3 | `### 子標題` | （H3） |
| 粗體 | `**很重要**` | **很重要** |
| 斜體 | `*強調*` | *強調* |
| 無序清單 | `- 項目一` | • 項目一 |
| 有序清單 | `1. 第一步` | 1. 第一步 |
| 連結 | `[賈伯斯數理教室](https://jobsedu.com.tw)` | [賈伯斯數理教室](https://jobsedu.com.tw) |
| 圖片 | `![alt 文字](/uploads/photo.jpg)` | （顯示圖片） |
| 表格 | 見下方範例 | 見下方 |
| 引用 | `> 學生回饋：上課很有趣` | （縮排引用樣式） |
| 程式碼（行內） | `` `const x = 1` `` | `const x = 1` |
| 程式碼（區塊） | 見下方範例 | 見下方 |
| 水平線 | `---` 自成一行 | ───────── |

### 表格範例

```markdown
| 科目 | 老師 | 時段 |
|---|---|---|
| 數學 | 王老師 | 週六上午 |
| 物理 | 林老師 | 週六下午 |
```

### 程式碼區塊範例

```markdown
​```python
def hello():
    print("Hi")
​```
```

> 補習班一般不太會用到程式碼區塊，除非寫資訊類部落格才用得上。

## 5. 圖片上傳建議

### 5.1 規格

| 用途 | 建議尺寸 | 最大檔案大小 | 格式 |
|---|---|---|---|
| 文章封面 | 1200 × 630 px | 500 KB | JPG（風景照）/ PNG（含文字） |
| 師資大頭照 | 800 × 800 px（正方形） | 300 KB | JPG |
| 課程封面 | 1600 × 900 px | 500 KB | JPG |
| LP hero | 1920 × 1080 px | 800 KB | JPG |
| 文章內圖 | 寬度 ≤ 1200 px | 400 KB | JPG / PNG / WebP |

### 5.2 命名規則

**檔名請用小寫英數 + 連字符**，避免中文與空格：

- 好：`teacher-lin-portrait.jpg`、`summer-2026-hero.jpg`
- 差：`林老師大頭.jpg`、`IMG 0123.jpg`、`照片(1).jpg`

中文檔名在某些瀏覽器與 CDN 上會出問題；空格會被網址轉成 `%20` 變得很醜。

### 5.3 alt 文字（無障礙與 SEO 必填）

每張圖片**一定**要寫 alt 描述。Decap 上傳時會跳出 alt 欄位。

- 好：`alt="林老師在白板上講解三角函數"`
- 差：`alt="圖片"`、`alt="DSC_0123"`、`alt=""`（空白等於沒寫）

### 5.4 壓縮工具建議

上傳前先壓縮，避免影響網站速度：

- [TinyPNG](https://tinypng.com/) — 最常用，免費，拖入即壓
- [Squoosh](https://squoosh.app/) — Google 出品，可調品質參數
- macOS 內建「預覽程式」：檔案 → 匯出 → 品質拉到 70%

> **建議**：1MB 以上的原始照片**一定要先壓縮**，否則手機開網站會 lag。

## 6. 品牌守則速查

完整版見 [BRAND_GUIDELINES.md](../BRAND_GUIDELINES.md)。撰寫前請花 30 秒掃過下表。

### 6.1 推薦詞彙（多用）

> 適合 / 協助 / 陪伴 / 累積 / 踏實 / 穩 / 個別化 / 量身規劃 / 我們發現 / 在我們的經驗裡 / 過去 13 年來

### 6.2 禁用詞彙（一定不要用）

> ✗ 保證上 X 大、100% 進步、絕對
> ✗ 業界第一、最強、最厲害
> ✗ 秒懂、開掛、無腦
> ✗ 親愛的家長 / 同學

### 6.3 Tone 範例對照

| 場景 | 不要這樣寫 | 改成這樣 |
|---|---|---|
| 開頭 | 親愛的家長您好！ | 給家長的一封信 |
| 招生 | 保證讓孩子數學突飛猛進！ | 我們陪伴孩子穩定累積數學基礎 |
| 師資 | 最強名師團隊 | 由台清交背景、13 年以上教學經驗的老師組成 |
| 試聽 | 立刻來體驗最棒的教學！ | 歡迎安排一次試聽，看課程是否適合 |

### 6.4 在地關鍵字（部落格 / 課程描述記得自然帶入）

> 嘉義 / 嘉義東區 / 康樂街 / 嘉中 / 嘉女 / 興華 / 協同 / 北興 / 民生

## 7. 發佈後檢查

每次發佈後請花 1 分鐘做以下檢查：

1. **等 1-2 分鐘**：Vercel 自動部署需要時間，不會立刻生效
2. **打開正式網站**：用無痕視窗或重新整理（Ctrl + Shift + R / Cmd + Shift + R），避免看到舊快取
3. **手機開一次**：補習班 70% 流量來自手機，自己用手機驗一遍
4. **檢查清單**：
   - [ ] 標題、摘要、發佈日是否正確
   - [ ] 圖片是否正確顯示（沒有出現破圖紅 X）
   - [ ] 內文格式是否符合預期（標題、清單、表格）
   - [ ] 連結是否點得開
   - [ ] 行動裝置上閱讀是否舒服

## 8. 常見問題

### Q1：我發現錯字怎麼改？

直接打開該篇文章 / 該位老師資料 → 改錯字 → 按「Publish」即可。1-2 分鐘後正式網站會更新。

### Q2：我不小心刪了一篇文章！

**先別緊張**，內容都在 Git 版本歷史裡。立刻聯絡工程師，告知大致時間（如「今天下午 3 點左右誤刪『學測倒數』那篇」），工程師可在 5-10 分鐘內救回來。

> **預防**：刪除前 Decap 會跳出確認視窗，請確認後再點。如果只是想下架不公開，把「草稿」開關打開即可，不必真的刪除。

### Q3：我發了文章，但網站上看不到？

依序排查：

1. 是否還是「草稿」狀態？打開該篇文章看「草稿」開關有沒有關閉
2. 是否剛發佈不到 2 分鐘？Vercel 部署需要時間
3. 是否快取問題？用無痕視窗或 Ctrl + Shift + R 強制重整
4. 仍看不到 → 聯絡工程師，附上文章標題與發佈時間

### Q4：圖片上傳很慢 / 失敗？

- 先檢查圖片是否超過 5MB（Decap 預設限制）。請先壓縮（見 §5.4）
- 換網路（Wi-Fi 不穩時換 4G 試試）
- 重新整理 Decap 後臺再試
- 仍失敗 → 聯絡工程師

### Q5：我可以排程在未來某天才發佈嗎？

Decap 預設不直接支援排程發佈。變通方案：

- 寫好內容後保持「草稿」開啟，到指定日期再手動關閉草稿並發佈
- 如需自動排程，請告知工程師評估加裝 plugin（屬於額外功能）

### Q6：手機上可以編輯嗎？

可以，Decap 支援手機瀏覽器。但富文字編輯在手機上操作較不便，**建議用桌機 / 筆電編輯**。手機只用來「看一下發佈結果」即可。

### Q7：可以多人同時編輯嗎？

技術上可以，但**強烈建議錯開**。同一篇文章兩人同時改，後存的會蓋掉先存的。多人協作建議：

- 每人負責不同集合（A 老師管課程、B 老師管部落格）
- 改之前在 LINE 群組吭一聲

### Q8：我想看以前的版本怎麼辦？

所有編輯都在 Git 版本歷史內。請聯絡工程師，告知文章標題與想回到的時間點，工程師可幫您比對與還原。

### Q9：標籤（tags）建議怎麼用？

標籤是給「相關文章推薦」用的。建議：

- 一篇文章 3-5 個標籤即可，不要太多
- 用詞統一（例：都用「學測」就別有些寫「學測」、有些寫「學科能力測驗」）
- 把常用的記在便利貼貼螢幕邊：`學測`、`會考`、`數學`、`物理`、`化學`、`英文`、`高三`、`國三`、`考前準備`、`學習方法`

### Q10：Decap 介面怎麼變英文了？

可能是登出後重新登入時瀏覽器語言判斷錯誤。右上角頭像 → Settings → Language 切回 `繁體中文（台灣）`。

## 10. 著陸頁（LP）快速複製 Playbook

> 對應 RFP §1.2.4「LP 快速複製機制」。
>
> 本章與 §3.6 互補：§3.6 教您在 **Decap 後臺** 點按鈕新增 LP；本章教的是 **直接複製檔案** 的工程做法。
> 兩條路殊途同歸（最終都產出 `src/content/landing/` 下的一個 `.md` 檔），差別只在介面 vs. 檔案。
> 不熟程式的負責人請走 §3.6；工程師或想批次開檔的進階使用者走本章較快。

### 10.0 目的

著陸頁（Landing Page，LP）是針對**單一行銷活動 / 檔期**的獨立專頁，例如：

- 寒假班、暑假班等季節檔期（如既有的 `summer-2026`）
- 學測 / 會考衝刺活動
- 特定廣告投放的到達頁（FB / Google Ads 點進來的第一頁）

LP 與一般課程頁不同之處：

- 版型固定（hero 大標 + 4 格賣點 + 活動內文 + 倒數計時 + 試聽表單），**不需要重新設計**，只要換文字。
- 路由為 `/lp/[campaign]`，**刻意不進 sitemap、不被搜尋引擎索引**（見 §10.4），適合搭配廣告短期使用。
- 活動結束後把 `published` 關掉即可下架，檔案保留供下次同檔期複製重用。

「快速複製」的精神就是：**複製一個既有 LP 檔 → 改幾個欄位 → 預覽 → 上線**，5 分鐘內開出一個新活動頁。

### 10.1 第 1 步 — 複製檔案

1. 進到著陸頁內容目錄：

   ```
   src/content/landing/
   ```

   目前裡面有一個範本可參考：`summer-2026.md`（2026 暑期班）。

2. 複製這個既有檔，改成新活動的檔名。例如要開「2026 寒假班」：

   ```bash
   # 在專案根目錄執行
   cp src/content/landing/summer-2026.md src/content/landing/winter-2026.md
   ```

   - 檔名規則：**小寫英數 + 連字符**，不要中文與空格（例：`winter-2026.md`、`gsat-sprint-2026.md`）。
   - 檔名本身不影響路由，路由由 frontmatter 的 `campaign` 欄位決定（見下一步），但**建議檔名與 `campaign` 取一致**，方便日後對照。

> 提醒：`summer-2026.md` 是「最完整的範本」，內文已經寫滿課程、學費、試聽說明等區塊。複製後可以直接在它的骨架上改字，比從空白檔開始快很多。

### 10.2 第 2 步 — 編輯 frontmatter

打開剛複製的新檔，最上方 `---` 與 `---` 之間就是 frontmatter（頁面設定）。

著陸頁的完整欄位定義在 `src/content/config.ts` 的 `landing` collection。逐欄說明如下：

| 欄位 | 必填 | 說明 | 範例 |
|---|---|---|---|
| `campaign` | ✅ | **URL 路由名**，會變成 `/lp/<campaign>`。小寫英數 + 連字符 | `winter-2026`（→ 網址 `/lp/winter-2026`） |
| `title` | ✅ | 瀏覽器分頁標題 / SEO 標題 / 麵包屑文字 | `2026 寒假班｜開學前最後衝刺` |
| `headline` | ✅ | hero 區第一眼的大標（H1） | `寒假，是補回上學期落差的黃金 30 天` |
| `subheadline` | 選填 | 大標下方的說明段落 | `1/20 - 2/15，數理英文小班，從盲點診斷到題型補強一次到位。` |
| `heroImage` | 選填 | hero 主視覺；放在內容目錄旁以相對路徑引用，或先留空 | `./winter-2026-hero.jpg`（暫不需要可整行刪除） |
| `ctaLabel` | 選填 | 行動按鈕文字（預設「立即試聽」） | `立即預約試聽` |
| `ctaHref` | 選填 | 按鈕連結（預設 `/contact`） | `/contact` 或 LINE 連結 |
| `startDate` | 選填 | 活動開始日，會顯示在 hero 的日期膠囊 | `2026-01-20` |
| `endDate` | 選填 | 活動結束日；未過期時頁面會顯示「活動結束倒數」計時器 | `2026-02-15` |
| `published` | 選填 | `true` = 對外可見並生成頁面；`false` = 不發佈（預設 `true`） | `true` |

> 注意：本專案 LP 用的欄位是上表這幾個，**沒有** `id`、`slug`、`hero.title` 之類的欄位——`campaign` 同時扮演「識別碼」與「URL slug」的角色，標題類文字則是 `title` / `headline` / `subheadline`。請以 `src/content/config.ts` 為唯一真實依據。

最少要改的兩個關鍵欄位：

1. **`campaign`** → 改成新活動的路由名（決定網址，務必唯一，不能和既有檔重複）。
2. **`title`／`headline`** → 改成新活動的文案。

以 `winter-2026.md` 為例，frontmatter 改完大致長這樣：

```yaml
---
campaign: winter-2026
title: 2026 寒假班｜開學前最後衝刺
headline: 寒假，是補回上學期落差的黃金 30 天
subheadline: 1/20 - 2/15，數理英文小班，從盲點診斷到題型補強一次到位。
ctaLabel: 立即預約試聽
ctaHref: /contact
startDate: 2026-01-20
endDate: 2026-02-15
published: true
---
```

接著把 `---` 以下的**內文**（課程結構、學費、試聽說明等）改成寒假班的內容即可。內文是標準 Markdown，語法見 [§4](#4-markdown-語法快速指引)，文案請對齊 [§6 品牌守則](#6-品牌守則速查)。

### 10.3 第 3 步 — 本地預覽

1. 在專案根目錄啟動開發伺服器：

   ```bash
   npm run dev
   ```

2. 打開瀏覽器，網址為 `localhost:4321/lp/<campaign>`。延續上例：

   ```
   http://localhost:4321/lp/winter-2026
   ```

3. 對照檢查：

   - hero 大標 / 副標 / 日期是否正確
   - 4 格賣點區（極小班、師資、教材、免費試聽）顯示正常
   - 內文排版（標題、清單、表格）無跑版
   - 若有設 `endDate` 且尚未到期，倒數計時器有出現
   - 底部試聽表單正常顯示
   - 用瀏覽器開發者工具切到手機寬度（390px）再看一次

> 改了 frontmatter 或內文後，dev server 通常會自動熱更新；若沒更新，重新整理頁面即可。新增整個檔案後若頁面 404，停掉 `npm run dev`（Ctrl + C）再重啟一次，讓 content collection 重新掃描。

### 10.4 第 4 步 — 上線注意事項

#### (a) getStaticPaths 會自動發現新檔 ✅

路由檔 `src/pages/lp/[campaign].astro` 的 `getStaticPaths()` 這樣寫：

```ts
const lps = await getCollection('landing', ({ data }) => data.published);
return lps.map((lp) => ({
  params: { campaign: lp.data.campaign },
  props: { entry: lp },
}));
```

意思是：**只要新檔放進 `src/content/landing/` 且 frontmatter `published: true`，build 時就會自動產生對應的 `/lp/<campaign>` 頁面**，不需要手動登記任何清單。這就是「快速複製」能成立的關鍵。

- 若 `published: false`，該頁不會被生成（等於下架）。
- `campaign` 必須在所有 LP 中唯一；若兩個檔用了相同 `campaign`，build 會因路由衝突而報錯。

#### (b) sitemap.xml 不會包含 LP 路由（刻意設計）⚠️

`astro.config.mjs` 的 sitemap 設定明確排除了 LP：

```js
sitemap({
  filter: (page) => !page.includes('/admin') && !page.includes('/lp/'),
}),
```

因此 **LP 頁面不會出現在 `sitemap.xml`，也就不會主動被 Google 索引**。這是刻意的：LP 是搭配廣告投放的短期到達頁，通常不希望它和正式課程頁在自然搜尋結果裡互相競爭、或在活動結束後留下死連結。

實務上代表：

- LP 的流量來源是**廣告 / LINE / QR code / 直接貼網址**，靠你主動把 `/lp/<campaign>` 連結分享出去，不是靠 SEO。
- 不需要、也不應該為了「讓 Google 找到 LP」去改這個 filter。
- 若哪天**真的**要讓某個 LP 進 sitemap 被索引（少見），那是工程決策，請開 issue 討論，不要逕自改 config（且本專案夜跑規則禁止改動 `astro.config.mjs` 的 site 設定）。

#### (c) 部署

確認預覽無誤後，照一般流程把改動 commit、push 到 feature branch，Vercel 會自動產生 preview 部署可供確認；正式上線（push 到 main / production）一律由業主白天拍板，不在夜間自動執行。

#### (d) 小結：複製一頁 LP 的最短路徑

1. `cp src/content/landing/summer-2026.md src/content/landing/<新campaign>.md`
2. 改 frontmatter 的 `campaign` / `title` / `headline`（+ 內文）
3. `npm run dev` → 開 `localhost:4321/lp/<新campaign>` 檢查
4. commit + push feature branch → preview 確認 → 交業主拍板上線

整個過程不需碰任何元件或設定檔，純粹複製 + 改 Markdown，符合「內容與程式分離」原則。
